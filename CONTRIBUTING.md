# Contributing to QRMeet

Thank you for your interest in contributing to QRMeet.

This project is distributed under the **MIT License**. See the `LICENSE` file for details.

---

## Agreement to Terms

By submitting a pull request or contributing to QRMeet in any form, you confirm that:

- You have read and understood these contribution guidelines.
- You agree that your contributions will be licensed under the same terms as the project (MIT License).
- You understand that your contributions may be accepted, modified, or rejected at the discretion of the maintainer.

---

## How to Contribute

### 1. Discuss Before Major Changes

For significant changes (new features, architectural modifications, or large refactors), please open an issue first to discuss your proposal. This helps ensure alignment with the project’s goals before development begins.

---

### 2. Reporting Issues

If you find a bug or have a feature request:

- Check existing issues to avoid duplicates.
- Open a new issue with:
  - A clear description of the problem or request
  - Steps to reproduce (for bugs)
  - Expected vs. actual behavior
  - Relevant environment details (if applicable)

---

### 3. Submitting Changes

To submit changes:

1. Fork the repository.
2. Create a new branch from the main branch.
3. Make your changes.
4. Ensure your code follows the project’s style and conventions.
5. Include tests if applicable.
6. Submit a pull request referencing the related issue.

---

### 4. Review Process

All contributions are subject to review.

- Not all proposals will necessarily be accepted.
- Maintainer feedback may be requested before merging.
- Final decisions regarding inclusion of changes are made by the project maintainer(s).

---

### 5. Contributors

Contributors whose work is merged into QRMeet may be listed in the `CONTRIBUTORS` file at the discretion of the maintainer.

Inclusion is typically based on meaningful contributions such as code, documentation, or other significant improvements that are accepted into the project.

---

### 6. Code Quality Guidelines

Please read [`docs/guidelines.md`](docs/guidelines.md) before contributing. It covers coding conventions, CSS rules, license requirements, commit message format, and legal compliance. It also explains how to configure an AI coding agent (e.g. Claude Code) to load these guidelines automatically.

Please ensure that:

- Code is clean and readable.
- Existing functionality is not broken.
- Documentation is updated when necessary.
- Commit messages are clear and descriptive.
